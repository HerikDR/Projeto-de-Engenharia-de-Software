# Projeto de Engenharia de Software: DevOps

## Sobre o Projeto: Sistema de Gerenciamento de Tarefas
Este é um sistema simples e prático para o gerenciamento de tarefas (To-Do List). Ele foi desenvolvido como um exercício acadêmico para aplicar na prática os conceitos de Desenvolvimento Ágil (Scrum e Kanban) e os fundamentos de DevOps, utilizando o ecossistema do GitHub.

### Objetivos Principais

- Aplicar Scrum definindo um Product Backlog e executando uma Sprint;
- Organizar o fluxo de trabalho utilizando Kanban através do GitHub Projects;
- Praticar a colaboração em código usando Issues, Branches e Pull Requests;
- Introduzir conceitos de Integração Contínua (CI) com automações no GitHub Actions.

## Funcionalidades do Sistema

#### 1. Cadastrar usuários;
#### 2. Criar novas tarefas;
#### 3. Editar tarefas existentes;
#### 4. Excluir tarefas desnecessárias;
#### 5. Listar todas as tarefas cadastradas;
#### 6. Marcar tarefas como concluídas.

## Linguagens e métodos utilizados
#### Backend: Java
#### Frontend: HTML e CSS
#### Banco de Dados: PostgreSQL
#### Gestão Ágil e Automação: GitHub (Projects, Issues e Actions)

## Product Backlog

1. Criar cadastro de usuário
2. Implementar login de usuário
3. Criar tarefa
4. Editar tarefa
5. Excluir tarefa
6. Listar tarefas
7. Marcar tarefa como concluída
8. Desenvolver interface frontend
